Academic Note on USER_ACCOUNT design: The correct ER-level pattern is a supertype/subtype (generalisation/specialisation hierarchy). USER_ACCOUNT is the supertype; PATIENT, DOCTOR, NURSE, EMPLOYEE are subtypes. Each 1:1 line represents total/partial participation. The relational mapping uses either (a) separate 1:1 FK columns in USER_ACCOUNT or (b) a shared PK between USER_ACCOUNT and each subtype table. The polymorphic profile_id + profile_type approach used in the schema mapping is valid for applications but is non-standard in academic relational theory — flag this to your supervisor if asked.
